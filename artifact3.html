Context
Problem & Solution
⚠ The Problem

Solar energy output is highly variable -- clouds, humidity, temperature, and sunlight hours all affect generation. Grid operators and energy planners need accurate forecasts to balance supply and demand. Traditional methods struggle with the non-linear relationships between weather variables and energy output.

✓ The Solution

Train, tune, and systematically compare multiple ML regression models on real weather datasets to find the best-performing algorithm for solar energy prediction. The goal: a model that is accurate, generalizable, and interpretable enough to be trusted in an energy planning context.

Reflection
Key Insights
🌿
No Single Best Algorithm -- Context Decides
SVR with an RBF kernel was the best-performing model (R² ≈ 0.528), outperforming both Random Forest and Linear Regression on this dataset. However, Linear Regression offered far better explainability. In energy planning contexts, interpretability matters as much as raw performance -- stakeholders need to trust and understand the model's predictions.
⚡
PCA Hurt More Than It Helped
Principal Component Analysis was tested as a dimensionality reduction step but degraded performance across all three models. The weather features in this dataset were individually meaningful -- compressing them into abstract components removed information the models needed. Direct feature selection outperformed PCA in every case.
📉
Overfitting Is a Real-World Problem
Models that performed well on training data often failed to generalize. Cross-validation and careful hyperparameter tuning were essential to building a model that would actually work on new weather data -- not just the data it had seen.
🔗
ML Bridges Engineering and Energy
This project directly connected my electrical engineering background with machine learning -- predicting physical system outputs from sensor data is exactly what ML does well. The same framework applies to predictive maintenance, fault detection, and control system optimization at Cummins.
