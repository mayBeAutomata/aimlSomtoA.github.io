Context
Problem & Solution
⚠ The Problem

The 8-tile puzzle has over 180,000 possible states. A brute-force search is computationally wasteful -- it explores too many dead ends and cannot guarantee finding the shortest solution path in a reasonable time. A smarter approach is needed.

✓ The Solution

A* search combines the actual cost of reaching a state with a heuristic estimate of the remaining distance to the goal. This guides the search intelligently, guaranteeing the optimal (shortest) solution while exploring far fewer states than uninformed methods.

Architecture
How A* Works
🏁
Initialize the Open List
Start with the initial puzzle state in a priority queue (open list). Assign it a cost score based on the heuristic function.
f(start) = 0 + h(start)
🎯
Select the Best Node
Always expand the state with the lowest f(n) score. This is what makes A* informed -- it prioritizes promising paths over arbitrary ones.
f(n) = g(n) + h(n)
🔀
Generate Successor States
Slide the blank tile in all valid directions (up, down, left, right) to generate neighboring states. Each move increments the actual path cost g(n) by 1.
📊
Evaluate with Heuristic
Score each successor using the chosen heuristic (Manhattan Distance or Misplaced Tiles). Add to the open list if not already explored with a lower cost.
✅
Goal Check & Path Reconstruction
When the goal state is dequeued, the algorithm terminates. Trace back through parent pointers to reconstruct the full optimal solution path.
4
Design Decision
Heuristic Functions Compared
Heuristic 1
Misplaced Tiles
Counts how many tiles are not in their goal position. Simple to compute but weaker -- it underestimates true distance because it ignores how far out of place each tile actually is.
Admissible · Less Informed
Heuristic 2 -- Preferred
Manhattan Distance
Sums the horizontal and vertical distance each tile must travel to reach its goal position. More informed than misplaced tiles -- explores fewer nodes and finds the solution faster while still guaranteeing optimality.
Admissible · More Informed ✓
5
Reflection
Key Insights
🧭
Heuristic Quality Determines Performance
Manhattan Distance dramatically outperforms Misplaced Tiles in nodes explored. Choosing a better heuristic -- one closer to true cost without overestimating -- is the single biggest lever for performance in informed search.
⚖️
Admissibility Is Non-Negotiable
A* only guarantees an optimal solution when the heuristic never overestimates the true remaining cost. Understanding why admissibility matters -- and how to verify it -- is a foundational AI engineering skill.
🌐
Search Space Structure Matters
The 8-puzzle has ~181,440 reachable states. Without intelligent search, exploring them all is computationally intractable. A* with Manhattan Distance typically finds optimal solutions in a fraction of those states.
🔗
Transferable to Real Engineering Problems
A* and its principles -- cost functions, state space search, heuristic design -- are directly applicable to robotics path planning, SLAM, game AI, and route optimization. This project built the intuition behind those real-world systems.
